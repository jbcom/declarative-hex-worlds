# PRD — `@jbcom/medieval-hexagon-gameboard` 1.0

> Authoritative scope for the v1.0.0 release. Folds in every finding from `.full-review/` (Phase 1-5) and gaps surfaced by the `standard-repo` profile audit. **Execution queue** lives in `.agent-state/directive.md` — this PRD is the *why*, the directive is the *what next*.

---

## 1. Product statement

`@jbcom/medieval-hexagon-gameboard` is a **deterministic, published-to-npm TypeScript library** for building turn-based games on KayKit's Medieval Hexagon asset pack. It ships:

- A Koota-ECS state model (traits, actions, queries)
- A hex-grid runtime built on `honeycomb-grid`, deterministic via `seedrandom`
- A scenario / blueprint / simulation engine
- A CLI (`medieval-hexagon-gameboard`) for headless validation, simulation, coverage, and asset ingest
- Optional React + Three.js bindings (peer-deps)
- A pre-baked FREE-edition KayKit asset manifest

1.0 is the **first public release**. Beyond this point, semver applies to every supported export, every CLI flag, and the determinism contract.

---

## 2. Quality bar for 1.0

The repo is unusually clean for its scale (Phase 1 verified: 0 `Math.random`, 0 `as any`, 0 `@ts-ignore`, 0 `TODO`, 0 stubs, 0 non-null assertions, no circular deps). 1.0 raises the bar in **eight dimensions** that the review found gaps in:

| Dimension | Today | 1.0 target |
|---|---|---|
| **Determinism contract** | Enforced by code review, not by tests | Cross-process byte-identical replay test gate |
| **Public API surface** | 37 subpath exports, ~10 leak internals | Tiered: ~13 supported subpaths; rest umbrella-only |
| **Error contract** | 130 raw `throw new Error()`, no taxonomy | `GameboardError` hierarchy in `src/errors.ts`, documented in `docs/api/errors.md` |
| **Bundle / parse cost** | umbrella consumers pay 22 KB gzip + ~395 KB raw for manifest they may not use | manifest is JSON + lazy-load; umbrella ≤ 12 KB gzip |
| **CLI cold start** | ~150-250 ms (entire library eagerly loaded) | ≤ 80 ms (dynamic per-subcommand imports) |
| **Security boundary on CLI inputs** | Path traversal in 40+ `--out*` sites; symlink-following walker | `safeResolveOutput()` enforced; walker rejects symlinks; prototype-pollution guard on JSON payloads |
| **CI gates** | Lint/typecheck/test/build/audit | + size-limit, cold-start bench, semgrep SAST, prod-only audit, SBOM, SLSA L3 attestation |
| **Standard-repo docs** | README + AGENTS + docs/pillars + docs/guides + TypeDoc | + CHANGELOG, STANDARDS, ARCHITECTURE, DESIGN, TESTING, DEPLOYMENT, STATE; frontmatter on every md |

## 3. Use cases for the runtime

This is the use-case enumeration step (per CLAUDE.md). Every architectural decision in the queue is anchored here.

| Use case | Consumer | Entry point | Latency budget | Identity-stability contract |
|---|---|---|---|---|
| **Headless scenario validation** | CI scripts, content authors | CLI `validate` / `validate-manifest` | ≤ 100 ms warm | Pure function over scenario JSON |
| **Headless coverage ledger** | CI scripts, release-please | CLI `coverage` (programmatic + JSON output) | ≤ 200 ms | Deterministic given fixed `--generatedAt` |
| **Headless simulation replay** | CI, save/load systems | CLI `simulate` + library `runSimulationStep` | ≤ 1 µs/step for sim-only workloads | Same seed + same script ⇒ byte-identical output across processes |
| **Browser gameplay (React)** | Game frontends | `@jbcom/medieval-hexagon-gameboard/react` + umbrella traits | per-frame budget 16 ms; selector hook ≤ 0.5 ms | Trait identity stable across subpaths (`splitting: true` invariant) |
| **Browser gameplay (Three)** | Custom Three.js renderers | `@jbcom/medieval-hexagon-gameboard/three` + umbrella | per-frame 16 ms | Same data ⇒ same GLTF URL set |
| **Asset ingestion** | Content tooling | CLI `extract` / `manifest` / library `ingest.ts` | one-shot, latency-bound by I/O | Same source tree ⇒ same manifest JSON (mod symlinks) |
| **Library-of-library embed** | Downstream npm packages | umbrella `'@jbcom/medieval-hexagon-gameboard'` | parse ≤ 30 ms | Tiered public API; semver-safe deprecations |

Tier rules that fall out of this matrix:

- **Headless use cases must NOT pay browser-bundling costs** → manifest-as-JSON, lazy CLI subcommands.
- **Browser use cases must NOT pay manifest cost unless they ask for it** → `freeManifest` off the umbrella.
- **All use cases share the determinism contract** → one cross-process test pins it.

## 4. Scope — what's in 1.0

Execution queue in `.agent-state/directive.md`. The directive enumerates every commit; this section enumerates the *outcomes* by epic.

### Epic A — Foundation gates (un-block everything else)

CI/tooling work that quantifies regressions for every subsequent fix. Six gates:

1. **`size-limit` budgets** on `dist/index.js`, `dist/cli.js`, `dist/manifest/free.js`, `dist/react.js`, `dist/three.js`.
2. **Coverage thresholds** in vitest (`v8` provider) — lines 80, branches 75, functions 80.
3. **`pnpm audit --prod --audit-level=high`** in the package CI job.
4. **Semgrep `p/owasp-top-ten` + `p/nodejs`** in CI; custom rule for `writeFileSync(resolve(parsed.flags.X), …)` without `safeResolveOutput`.
5. **Biome rule set tightened** (see Appendix A).
6. **`pnpm.overrides`** clears the two `pnpm audit` moderates (`yaml >=2.8.3`, `brace-expansion >=5.0.6`).

### Epic B — Performance criticals (publish-blocking)

- **B1/B2**: `freeManifest` → JSON + lazy loader; removed from umbrella. (Saves 22 KB gzip / 30-150 ms parse per umbrella consumer.)
- **B3**: `cli.ts` (4,297 LOC) → `src/cli/index.ts` + `src/cli/commands/*.ts` registry with dynamic imports; `node:util.parseArgs`. (Cuts CLI cold start from ~200 ms → ~40 ms.)
- **B4-B8**: Hot-path wins — `tilesByKey` materialized at projection; single-pass actor-targets reducer; `tryParseHexKey`; React selector hash-stabilization; `structuredClone` over `JSON.parse(JSON.stringify(…))`.

### Epic C — Security criticals (publish-blocking)

- **C1**: `safeResolveOutput(value, outRoot=cwd())` enforced on all 40+ CLI `--out*` write sites. `extract` requires `--force` for destructive paths and refuses non-empty pre-existing dirs.
- **C2**: `listFiles` rejects symlinks and verifies `realpathSync` boundary; cycle-safe.
- **C3**: Prototype-pollution guard on `readPieceSourceRoots`; `Object.create(null)` outputs; JSON reviver strips `__proto__` / `constructor` / `prototype`.
- **C4-C7**: `sh -c` quoting fix; relative-path error messages; debug-flag-gated stack traces; no source-map publish.

### Epic D — Architectural debt (publish-blocking)

- **D1**: Subpath export tiering — ~10 internal subpaths demoted from `package.json#exports` and `tsup.config.ts` entries; retained as umbrella re-exports. (Tier list in Appendix B.)
- **D2**: `src/errors.ts` taxonomy — 130 throws migrated to typed errors.
- **D3**: `simulation.ts` (5,213 LOC) decomposed into `simulation/{engine,script,report,assertions,index}.ts` with `assertNever` exhaustiveness. *Note: Phase 1 H-3 "double-dispatch" was corrected by Phase 2 P-perf — it's not a runtime double-dispatch, only a decomposition / drift-risk concern.*
- **D4**: `createKayKitGuideScenarios` (377-line function) inverted to data table + iteration.
- **D5**: `src/traits.ts` + `src/actions.ts` umbrellas (discoverability fix).
- **D6**: React/Three peer-dep runtime guards.
- **D7**: Package-scoped scripts moved into `packages/medieval-hexagon-gameboard/scripts/`.
- **D8/D9/D10**: Audit-script extraction (`scripts/_lib.ts`); `audit-workspace.ts` split into `scripts/audits/`; `smoke-packed-consumer.ts` split into runtime smoke + compile-time API attestation with labelled-phase harness.

### Epic E — Test debt (publish-blocking)

- **E1**: Cross-process determinism gate (spawn N subprocesses, byte-identical assertion).
- **E2**: Public API snapshot gate.
- **E3**: CLI hostile-input tests (path traversal, symlink loop, prototype payload).
- **E4**: Trait identity test across subpaths.
- **E5**: CLI cold-start bench.
- **E6**: Simulation throughput micro-bench.
- **E7**: React render-count assertion for memoized selector hooks.
- **E8**: Coverage thresholds enforced (gate from A8).

### Epic F — Documentation (publish-blocking)

- **F1d-F5d**: README quickstart, CLI reference, determinism guide, peer-deps guide, errors API. (Phase 3b top-5.)
- **F6d-F12d**: CHANGELOG, STANDARDS, ARCHITECTURE, DESIGN, TESTING, DEPLOYMENT, STATE. (Standard-repo profile.)
- **F13d**: Frontmatter on every md.

### Epic G — Release readiness (final gate)

- **G2**: `@cyclonedx/cyclonedx-npm` SBOM release artifact.

## 5. Out of scope for 1.0

(Conscious cuts — *not* "future work" euphemisms.)

- **PAID-edition asset manifest.** The PRD ships FREE only. PAID is a same-shape addition; deferring keeps the 1.0 surface tight.
- **Multi-edition manifest negotiation API.** Until a second edition exists, single-edition shape stays.
- **Server-authoritative simulation (Node-cluster fan-out).** Determinism contract enables it; no consumer asked for it yet.
- **Network protocol / save-game format.** Determinism + structured-clone snapshot is enough for in-process save-load; a portable wire format would be a separate package.
- **Mobile bundles (Capacitor / RN).** Outside profile scope.

---

## 6. Non-negotiables (invariants the 1.0 build must preserve)

These are the values 1.0 commits to. Any future change has to test against these or revise this document.

1. **Determinism.** Identical inputs (seed + scenario + script) produce byte-identical outputs across processes and platforms. Pinned by E1.
2. **No `Math.random`, `crypto.randomUUID`, `Date.now`, `performance.now`** in `packages/medieval-hexagon-gameboard/src/` — *except* in CLI-output formatters with explicit override flags (and the single `cli.ts:2296` site falls under that exception).
3. **No `any`, no `@ts-ignore`, no `as any`, no non-null assertions** in src/ or scripts/. Biome enforces.
4. **Zero `TODO`, `FIXME`, `it.todo`, empty bodies, stub markers.**
5. **No circular dependencies** between modules.
6. **`splitting: true`** stays on; trait identity test (E4) keeps it honest.
7. **ESM only, Node ≥22, sideEffects: false.**
8. **Every CI gate runs locally via `pnpm verify`.**
9. **Public API tier table** in `docs/api/public-api.md` is source-of-truth; `package.json#exports` mirrors it.

## 8. Risk register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Subpath demotion (D1) breaks an unknown external consumer | Low (pre-1.0, no consumers) | Medium | This is the only chance to make this change. Document tier table prominently. Issue major-version commitment in CHANGELOG. |
| Manifest JSON conversion (B1) changes byte layout | Low | High | Cross-process determinism test (E1) catches drift; CI drift check on regenerated manifest. |
| CLI refactor (B3) drops a flag silently | Medium | High | `smoke-built-cli.ts` (1,214 LOC) exercises the surface; expand it before refactor. |
| Symlink rejection (C2) breaks a legitimate developer workflow | Low | Medium | Document and provide `--allow-symlinks` opt-in only if a real user reports it. |
| Trait identity breaks under subpath demotion (D1) | Medium | High | E4 trait-identity test gates D1. |
