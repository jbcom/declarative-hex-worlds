# PRD — `declarative-hex-worlds` 1.0

> Authoritative scope for the v1.0.0 release. Folds in every finding from `.full-review/` (Phase 1-5) and gaps surfaced by the `standard-repo` profile audit. **Execution queue** lives in `.agent-state/directive.md` — this PRD is the *why*, the directive is the *what next*.

---

## 1. Product statement

`declarative-hex-worlds` is a **deterministic, published-to-npm TypeScript library** for building turn-based games on KayKit's Medieval Hexagon asset pack. It ships:

- A Koota-ECS state model (traits, actions, queries)
- A hex-grid runtime built on `honeycomb-grid`, deterministic via `seedrandom`
- A scenario / blueprint / simulation engine
- A CLI (`declarative-hex-worlds`) for headless validation, simulation, coverage, and asset ingest
- First-class React + Three.js bindings (hard runtime `dependencies`, not optional peers)
- A pre-baked FREE-edition KayKit asset manifest

1.0 is the **first public release**. Beyond this point, semver applies to every supported export, every CLI flag, and the determinism contract.

## 2a. Asset model: bootstrap, not bundle (2026-05-26 correction)

The library does NOT ship the KayKit asset tree. The tarball ships:

- The TypeScript runtime (`dist/`).
- The CLI (`dist/cli.js`).
- The **manifest as JSON metadata** — what the consumer's asset tree should look like after bootstrap.
- Schemas / loaders / validators.

Asset acquisition is consumer-driven via the CLI's **`bootstrap`** subcommand (and a programmatic mirror, `bootstrapKayKitAssets({...})`). Behaviors:

1. **Default target**: `public/assets/models/` — heuristically detected (`public/assets/`, `public/`, `static/`, `assets/`, `<cwd>`) or explicit via `--out <dir>`.
2. **Source selection**:
   - `--source github` (default for FREE): clones / sparse-checkouts `https://github.com/KayKit-Game-Assets/KayKit-Medieval-Hexagon-Pack-1.0.git`, extracts only the `Assets/gltf/` subtree.
   - `--zip <path>`: accepts a zip file (FREE or EXTRA from itch.io); auto-detects edition by inspecting the archive's manifest/marker files.
3. **Layout**: mirrors upstream structure exactly under `<out>/addons/kaykit_medieval_hexagon_pack/Assets/gltf/...`. The library reads addons via that path at runtime.
4. **Filter**: gltf files only. Skip `*.fbx`, `*.obj`, `*.mtl`, `*.fbm/`, `__MACOSX/`. Optional `--include-source-formats` opt-in for power users.
5. **Integrity**: write `<out>/addons/kaykit_medieval_hexagon_pack/.bootstrap.json` capturing source URL/zip, SHA256 of each extracted file, total count, edition, library version, and a timestamp. `bootstrap --verify` re-checks integrity without re-downloading.
6. **Idempotency**: `bootstrap` is safe to re-run; existing files are checksum-compared, drift produces a clear diff report; `--force` re-fetches.
7. **Determinism**: same source (commit SHA / zip checksum) ⇒ same tree, byte-identical.

Tests exercise both shapes:
- **Unit**: zip parsing, gltf filtering, layout mirroring, integrity report shape (no network).
- **Integration**: end-to-end with `references/KayKit_Medieval_Hexagon_Pack_1.0_FREE/` fed through `--zip` path.
- **E2E**: full `bootstrap` → runtime asset resolution → render path, both `--zip` and `--source github` (the GitHub variant runs in scheduled CI only — not on every PR — to bound rate).

## 3. Use cases for the runtime

This is the use-case enumeration step (per CLAUDE.md). Every architectural decision in the queue is anchored here.

| Use case | Consumer | Entry point | Latency budget | Identity-stability contract |
|---|---|---|---|---|
| **Headless scenario validation** | CI scripts, content authors | CLI `validate` / `validate-manifest` | ≤ 100 ms warm | Pure function over scenario JSON |
| **Headless coverage ledger** | CI scripts, release-please | CLI `coverage` (programmatic + JSON output) | ≤ 200 ms | Deterministic given fixed `--generatedAt` |
| **Headless simulation replay** | CI, save/load systems | CLI `simulate` + library `runSimulationStep` | ≤ 1 µs/step for sim-only workloads | Same seed + same script ⇒ byte-identical output across processes |
| **Browser gameplay (React)** | Game frontends | `declarative-hex-worlds/react` + umbrella traits | per-frame budget 16 ms; selector hook ≤ 0.5 ms | Trait identity stable across subpaths (`splitting: true` invariant) |
| **Browser gameplay (Three)** | Custom Three.js renderers | `declarative-hex-worlds/three` + umbrella | per-frame 16 ms | Same data ⇒ same GLTF URL set |
| **Asset ingestion** | Content tooling | CLI `extract` / `manifest` / library `ingest.ts` | one-shot, latency-bound by I/O | Same source tree ⇒ same manifest JSON (mod symlinks) |
| **Library-of-library embed** | Downstream npm packages | umbrella `'declarative-hex-worlds'` | parse ≤ 30 ms | Tiered public API; semver-safe deprecations |

Tier rules that fall out of this matrix:

- **Headless use cases must NOT pay browser-bundling costs** → manifest-as-JSON, lazy CLI subcommands.
- **Browser use cases must NOT pay manifest cost unless they ask for it** → `freeManifest` off the umbrella.
- **All use cases share the determinism contract** → one cross-process test pins it.

## 4. Scope — what's in 1.0

Execution queue in `.agent-state/directive.md`. The directive enumerates every commit; this section enumerates the *outcomes* by epic.

### Epic R — Restructure (precedes everything else)

The monorepo shape is wrong. The library is a **single package** that should live at the repo root, with `src/` decomposed into **domain sub-packages** that talk to each other only through barrel public APIs. SimpleRPG is **not** consumer-facing — it's a test fixture for integration + e2e. Three deliverables:

1. **De-monorepo.** Delete `pnpm-workspace.yaml`; move `{src,tests,*.config.ts,tsconfig.json,tsup.config.ts}` to repo root. Merge `package.json` files (single `package.json` at root with the `declarative-hex-worlds` name + dependencies + exports + scripts). Drop Nx (we have one package, no graph to coordinate). Drop `apps/docs/` workspace entry — keep `docs/` content + vitepress config but don't register as a workspace package.
2. **Decompose `src/`.** Today's 38 flat modules collapse into ~13 domain sub-packages, each with its own `index.ts` barrel and TSDoc-tagged public surface. Cross-domain imports MUST traverse the barrel, never reach into a sibling's files. Biome `noRestrictedImports` enforces. See Appendix C for the target layout.
3. **Relocate SimpleRPG to tests.** `examples/simple-rpg-usage.ts` (and the `examples/*.json` SimpleRPG fixtures) move into `tests/integration/simple-rpg/` (unit-level fixture + assertions) and `tests/e2e/simple-rpg/` (playwright happy-path + failure cases). The bundled examples published in 0.x are removed from the `exports` map; the `examples` directory in the published tarball drops to `examples/blueprint-board-usage` only (or, more likely, also moves to tests).

This block precedes Epic A-G because every downstream epic addresses paths that change shape. After Epic R lands, the directive items in Epics B-G are re-pointed at the new paths.

### Epic A — Foundation gates (un-block everything else)

CI/tooling work that quantifies regressions for every subsequent fix. Six gates:

1. **Manifest integrity gate** — `pnpm assets:free && git diff --exit-code src/manifest/free.ts` proves the bundled manifest matches `assets/free/manifest.json`. (Replaces the size-limit gate from an earlier draft. Bundle-size budgets are **rejected**: this library *bundles* the FREE KayKit pack so consumers get a working game out-of-box — punishing the bundle weight contradicts the product. A runtime warm-start bench is added in its place, non-blocking initially.)
2. **Coverage thresholds** in vitest (`v8` provider) — **100 % statements / 100 % branches / 100 % functions / 100 % lines**. Hard fail. (Phase 1 baseline: 86.2 / 76.5 / 93.2 / 85.9 — the gap closes during Epic E and the renderer-binding visual epic.)
3. **`pnpm audit --prod --audit-level=high`** in the package CI job.
4. **Semgrep `p/owasp-top-ten` + `p/nodejs`** in CI; custom rule for `writeFileSync(resolve(parsed.flags.X), …)` without `safeResolveOutput`.
5. **Biome rule set tightened** (see Appendix A).
6. **`pnpm.overrides`** clears the two `pnpm audit` moderates (`yaml >=2.8.3`, `brace-expansion >=5.0.6`).

### Epic B — Performance criticals (publish-blocking)

- **B1 (re-scoped)**: `src/manifest/free.ts` becomes **autogenerated** from `assets/free/manifest.json` (the existing source-of-truth produced by `pnpm assets:free`). Banner + real `generatedAt`. CI drift check via A3b. This fixes Phase 1 H-1 *without* punishing the umbrella consumer.
- **B2 (rejected)**: removing `freeManifest` from the umbrella. **The bundled manifest is the product.** Out-of-box consumers MUST get `import { freeManifest } from 'declarative-hex-worlds'`. Replaced by **B2b** which adds an additional `loadFreeManifest(): Promise<…>` for explicit-lazy use cases — both shapes ship.
- **B3**: `cli.ts` (4,297 LOC) → `src/cli/index.ts` + `src/cli/commands/*.ts` registry with dynamic imports; `node:util.parseArgs`. (Cuts CLI cold start from ~200 ms → ~40 ms.) **Headless paths** (`validate`, `coverage`) no longer pay manifest cost; **render paths** still get the full bundle via umbrella.
- **B4-B8**: Hot-path wins — `tilesByKey` materialized at projection; single-pass actor-targets reducer; `tryParseHexKey`; React selector hash-stabilization; `structuredClone` over `JSON.parse(JSON.stringify(…))`.

### Epic C — Security criticals (publish-blocking)

- **C1**: `safeResolveOutput(value, outRoot=cwd())` enforced on all 40+ CLI `--out*` write sites. `extract` requires `--force` for destructive paths and refuses non-empty pre-existing dirs.
- **C2**: `listFiles` rejects symlinks and verifies `realpathSync` boundary; cycle-safe.
- **C3**: Prototype-pollution guard on `readPieceSourceRoots`; `Object.create(null)` outputs; JSON reviver strips `__proto__` / `constructor` / `prototype`.
- **C4-C7**: `sh -c` quoting fix; relative-path error messages; debug-flag-gated stack traces; no source-map publish.

### Epic D — Architectural debt (publish-blocking)

- **D1 (re-scoped)**: Subpath tiering by **documentation**, not by demotion. All current subpaths remain published — this is an asset-bundled library where consumers may legitimately reach into internals for custom rendering, data inspection, or modding. Instead, `docs/api/public-api.md` defines three tiers (Stable / Supported-for-extension / Internal-but-exposed), and TSDoc `@public` / `@beta` / `@internal` tags reflect the same. Internal-tier symbols may still receive semver-bumping changes with notice. The original "demote ~10 subpaths" proposal punished mod authors and is rejected.
- **D2**: `src/errors.ts` taxonomy — 130 throws migrated to typed errors.
- **D3**: `simulation.ts` (5,213 LOC) decomposed into `simulation/{engine,script,report,assertions,index}.ts` with `assertNever` exhaustiveness. *Note: Phase 1 H-3 "double-dispatch" was corrected by Phase 2 P-perf — it's not a runtime double-dispatch, only a decomposition / drift-risk concern.*
- **D4**: `createKayKitGuideScenarios` (377-line function) inverted to data table + iteration.
- **D5**: `src/traits.ts` + `src/actions.ts` umbrellas (discoverability fix).
- **D6 (REJECTED)**: peer-dep runtime guards. Per user direction 2026-05-26, react/three/react-dom move from `peerDependencies` to `dependencies` — the library is unusable without its bindings, exactly like koota. No guards needed; consumer **always** has them. Replaces with **D6b** (umbrella re-exports `react/` and `three/` like every other sub-package).
- **D7**: Package-scoped scripts moved into `scripts/`.
- **D8/D9/D10**: Audit-script extraction (`scripts/_lib.ts`); `audit-workspace.ts` split into `scripts/audits/`; `smoke-packed-consumer.ts` split into runtime smoke + compile-time API attestation with labelled-phase harness.

### Epic E — Test debt (publish-blocking, raises floor to 100 %)

- **E0**: **Close every coverage gap to 100 %.** Per Phase 1 baseline, the worst-covered files are `simulation.ts` 67.7 %, `patrol.ts` 72.3 %, `recipe.ts` 78.9 %, `commands.ts` 82.8 %, `systems.ts` 83.3 %; the worst branch coverage is `world-rules.ts` 53.3 % and `examples/simple-rpg-usage.ts` 53.8 %. - **E2**: Public API snapshot gate.
- **E3**: CLI hostile-input tests (path traversal, symlink loop, prototype payload).
- **E4**: Trait identity test across subpaths.
- **E5**: CLI cold-start bench.
- **E6**: Simulation throughput micro-bench.
- **E7**: React render-count assertion for memoized selector hooks.
- **E8**: Coverage thresholds enforced at **100 / 100 / 100 / 100** in vitest config; CI gates.
- **E9**: **Visual integration confirmation.** Every `react.ts` / `three.ts` / `examples/` exported binding has a vitest-browser test that renders into a real Chromium and asserts against a committed PNG screenshot snapshot. 4. **Zero `TODO`, `FIXME`, `it.todo`, empty bodies, stub markers.**
5. **No circular dependencies** between modules.
6. **`splitting: true`** stays on; trait identity test (E4) keeps it honest.
7. **ESM only, Node ≥22, sideEffects: false.**
8. **Every CI gate runs locally via `pnpm verify`.**
9. **Public API tier table** in `docs/api/public-api.md` is source-of-truth; `package.json#exports` mirrors it.

## Appendix B — Public API tier table (D1, re-scoped)

All subpaths remain published. Tiering is **documentary** — TSDoc-tagged and surfaced in `docs/api/public-api.md`.

**Tier 1 — Stable (semver-strict, breaking-change = major):**

`.` (umbrella), `./react`, `./three`, `./cli`, `./manifest/schema`, `./manifest/free`, `./scenario`, `./blueprint`, `./gameboard`, `./recipe`, `./coverage`, `./compatibility`, `./examples/*`. These are the supported consumer entry points; intentional breaking changes require major version + migration guide.

**Tier 2 — Supported-for-extension (semver-strict but smaller surface contract):**

`./koota`, `./runtime`, `./actors`, `./movement`, `./patrol`, `./quests`, `./pieces`, `./layout`, `./projection`, `./navigation`, `./occupancy`, `./validation`. Stable enough for mod authors and custom-renderer consumers to depend on. Breaking changes documented with migration paths.

**Tier 3 — Internal-but-exposed (semver-permissive, label "@internal" in TSDoc):**

`./coordinates`, `./grid`, `./selectors`, `./systems`, `./commands`, `./registry`, `./rules`, `./rule-types`, `./world-rules`, `./interop`, `./ingest`, `./catalog`, `./types`. Useful for deep modding; may change between minor versions with a CHANGELOG note. Consumers who pin these accept the support tradeoff.

Trait identity safety: regardless of tier, `splitting: true` in tsup ensures the same trait identity across subpaths. Pin via the `E4` trait-identity test.

## Appendix C — Target `src/` decomposition (Epic R)

The shape follows **idiomatic koota app layout** (see `~/src/reference-codebases/koota/examples/cards/src` and `examples/n-body-react/src`): traits live in `traits/`, systems in `systems/`, action factories in `actions.ts`, world bootstrap in `world.ts`, lifecycle composition in `frameloop.ts`/`startup.ts`. Each is a **domain-aligned set of files**, not one ECS blob.

Notes vs the earlier draft:

- **No `koota/` sub-package.** That conflated *the dep `koota`* with *our integration of it*. Koota is consumed; the integration shape it expects (traits/, systems/, actions.ts, world.ts) is what we mirror.
- **`runtime/` flattened.** Coordinate work splits naturally into `coordinates/`; gameboard/occupancy/navigation belong together under `gameboard/`; pieces get their own home; nothing left over for a "runtime" name.
- **`actors/` → `traits/actors.ts` + `systems/movement-system.ts` etc.** Per koota conventions, traits are flat declarations; systems read traits and act. Splitting along subsystem (movement, patrol, quests) and concern (trait file vs system file) matches how koota itself documents app structure.
- **`commands.ts` + `selectors.ts` are `@internal`** — they're plumbing for `actions.ts` and React hooks, not public surface.
- **`world.ts`, `actions.ts`, `frameloop.ts`, `startup.ts`** sit at the `src/` root because they're the composition layer — they orchestrate everything else and need to see every sub-package.

**Public-surface rule (Biome `noRestrictedImports`):** within `src/`, the only allowed import path from another sub-package is `'./<sibling-name>'` (the barrel). Never `'./<sibling-name>/some-internal.js'`. Test files in `tests/` may reach internals **only** through documented `tests/internal/` re-exports so the contract is explicit.
